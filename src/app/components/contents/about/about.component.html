Why use Scouter?

Scouter is a free and open source stream processing tool to contextualize events. Scouter makes it easy to reliably process streams of data, it was developed to be generic, fully configurable and extensible.

Scouter has many use cases, it is a generic tool that helps in capturing, analyzing, scoring and storing the contextual information of a given application domain. The processing depends on a semantic-based approach that exploits ontologies to score the relevancy of contextual information.

Scouter integrates with the Kafka queueing system and MongoDB database . A Scouter workflow consumes streams of data and processes those streams using Kafka Streams, repartitioning the streams between each stage of the computation however needed. Read more in the documentation.


How to use Scouter ?

In order to use Scouter, a web interface is used to build a workflow and to configure different components. This workflow comes with a default flow option that connects all components together. If you want to create a new workflow, you start by defining a geo-location area on a map , this will create a bounding box for the data to be fetched, adding geo-filtering to our system.

Adding a data source simply requires a drag and drop interaction and some configuration. NLP functions are applied to the streams for scoring purpose according to the relevancy of the event to detect. This scoring is based on annotating the texts. In addition, the system provides several analytics tools such as topic extraction and sentimental analysis to improve annotations.
